# Backend Test Data

Backend fixtures and generated Chan-analysis results used by Mist tests and
frontend sync scripts.

## Directory Structure

```
test-data/
├── fixtures/
│   ├── k-line/         # Raw K-line inputs for Chan regression tests
│   └── patterns/       # Small pattern fixtures used by unit tests
└── test-results/
    ├── raw/            # Generated JSON results from Chan regression tests
    └── types/          # Generated TypeScript definitions for frontend sync
```

## Usage

```typescript
import { shanghaiIndexData2024_2025 } from '@test-data/fixtures/k-line/shanghai-index-2024-2025.fixture';
import { KLineFixtures } from '@test-data/fixtures/patterns/k-line-fixtures';
```

## Syncing

```bash
# Generate Chan results, then sync them to the frontend repo
pnpm run test:chan:shanghai-2024-2025
pnpm run test:gen-types
pnpm run test:sync
```

## Files

### Fixtures

- `fixtures/k-line/shanghai-index-2024.fixture.ts`
- `fixtures/k-line/shanghai-index-2024-2025.fixture.ts`
- `fixtures/k-line/csi300-2025.fixture.ts`
- `fixtures/patterns/k-line-fixtures.ts`

### Generated Results

- `test-results/raw/` - JSON result files generated by selected Chan tests
- `test-results/types/` - TypeScript definitions generated from raw results

## Development

The test data system provides:
- Backend imports through the `@test-data` Jest/TypeScript alias
- Stable raw K-line inputs for regression tests
- Generated result JSON and type definitions for frontend sync
